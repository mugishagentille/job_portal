🎯 1. Project Overview

JobPortal is a comprehensive web-based job application management system that connects employers with job seekers. It provides a seamless platform for employers to post jobs, manage applications, and for job seekers to find and apply for jobs that match their skills and interests.

Project Objectives

  • Create a centralized platform for job posting and application management
  • Enable employers to post jobs and manage applications efficiently
  • Allow job seekers to browse jobs, apply, and track application status
  • Implement secure authentication with role-based access control
  • Provide admin with full system oversight and reporting capabilities
  • Deliver a responsive, user-friendly interface accessible on all devices

❓ 2. Problem Statement

In today's competitive job market, both employers and job seekers face significant challenges:

  • For Employers: Difficulty managing applications, tracking candidates, and finding qualified talent
  • For Job Seekers: Hard to find relevant jobs, track applications, and stand out to employers
  • Lack of Centralization: No single platform for both job posting and application tracking
  • Manual Processes: Many employers still use email or spreadsheets to manage applications
  • No Status Tracking: Candidates don't know where they stand in the hiring process

Our Solution: JobPortal addresses these challenges by providing a unified platform where employers can post jobs and manage applications, while job seekers can find relevant opportunities and track their application status in real-time.

💡 3. Solution Overview

JobPortal is a complete job management ecosystem with three distinct user roles:

👑 Admin

  • Full system control and oversight
  • User management (CRUD)
  • Job management (approve/reject)
  • Category management
  • View all applications and reports
  • Revenue and activity analytics

🏢 Employer

  • Post new jobs with detailed descriptions
  • Edit and manage own job postings
  • View applications for their jobs
  • Update application status (pending, reviewed, shortlisted, hired, rejected)
  • View applicant resumes and details
  • Track job performance statistics

👤 Job Seeker

  • Browse and search jobs by category, location, type
  • Apply for jobs with cover letter and resume upload
  • Track application status
  • Save favorite jobs to wishlist
  • View application history
  • Update profile and resume

🗄️ 6. Database Design

The system uses 6+ tables with proper relationships and foreign keys.

📸 Admin Dashboard Overview

Table Structure

Table Description Key Fields
users User accounts & authentication id, full_name, email, password, role, status
categories Job categories id, name, icon, description, status
jobs Job postings id, employer_id, title, description, category_id, location, salary, job_type, status
applications Job applications id, job_id, applicant_id, cover_letter, resume_url, status
saved_jobs User saved jobs id, user_id, job_id

Entity Relationship Diagram (ERD)

  • Users → Jobs (one-to-many) - Employer relationship
  • Users → Applications (one-to-many) - Applicant relationship
  • Jobs → Applications (one-to-many) - Job applications
  • Jobs → Categories (many-to-one) - Category classification
  • Users → Saved Jobs (many-to-many) - Wishlist functionality

⚙️ 7. Key Functionalities by Role

Admin CRUD Operations

  • ✅ Create: Add new users, categories, jobs
  • ✅ Read: View all users, jobs, applications, reports
  • ✅ Update: Edit user roles, job details, category information
  • ✅ Delete: Remove users, jobs, categories (with confirmation)
  • ✅ Activate/Deactivate: Toggle user and job status

Employer Functionalities

  • Post Jobs: Create job listings with title, description, requirements, salary, location, job type
  • Manage Jobs: Edit, update, and delete own job postings
  • View Applications: See all applicants for each job
  • Update Status: Change application status (Pending → Reviewed → Shortlisted → Hired/Rejected)
  • Download Resumes: Access applicant resumes and cover letters
  • Analytics: Track applications per job and performance

Job Seeker Functionalities

  • Browse Jobs: Search and filter by category, location, job type, salary
  • Apply for Jobs: Submit applications with cover letter and resume upload
  • Track Applications: View status of all submitted applications
  • Save Jobs: Save interesting jobs to wishlist for later
  • Profile Management: Update profile information and upload new resume
  • View Application History: See all jobs applied for and their status

🔒 9. Security Features

  • ✅ Password Hashing: All passwords stored using password_hash() with BCRYPT algorithm
  • ✅ Prepared Statements: All database queries use PDO prepared statements to prevent SQL injection
  • ✅ Session Management: Secure session handling with session_regenerate_id()
  • ✅ Role-Based Access Control: Each user role has specific permissions (requireAdmin, requireEmployer, requireLogin)
  • ✅ XSS Protection: All user input sanitized with htmlspecialchars()
  • ✅ Form Validation: Both client-side and server-side validation
  • ✅ Status Checks: Inactive users cannot access the system
  • ✅ File Upload Security: Resumes validated for type and size
